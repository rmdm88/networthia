# Networthia

Networthia — приложение для учета личного капитала (net worth) по методу срезов.
Вместо микроменеджмента расходов вы периодически фиксируете остатки по счетам на выбранную дату, а сервис показывает динамику капитала и структуру активов.

## Ключевая идея

- Нет трекинга каждой транзакции и категорий трат.
- Нет обязательных банковских интеграций.
- Есть только контрольные точки состояния капитала: дата + балансы по счетам.
- Все расчеты на дашборде приводятся к RUB для единой аналитики.

## Что умеет приложение

- Дашборд: общий капитал, дельта к предыдущему срезу, динамика во времени, распределение по счетам, топ-изменения, риск-метрики и фильтрация по тегам.
- Управление счетами: создание, редактирование, выбор валюты, мультивыбор тегов, архивирование и drag-and-drop сортировка.
- Срезы: быстрый табличный ввод балансов, автоматическая подгрузка курсов на дату через `/api/fx`, удаление срезов.
- Данные: экспорт состояния в JSON и импорт JSON с полной заменой локального состояния.

## Как пользоваться (MVP-сценарий)

1. Перейдите в раздел `Счета` и создайте список ваших счетов.
2. Для каждого счета выберите валюту и нужные теги.
3. Перейдите в `Срезы` и добавьте первый срез на нужную дату.
4. Введите текущие остатки по всем активным счетам.
5. Добавляйте новые срезы раз в неделю или месяц.
6. Анализируйте дашборд: общая динамика, вклад счетов, ключевые изменения.

## Хранение данных и приватность

- Данные хранятся в `localStorage` браузера.
- Ключ хранилища: `capital-atlas-state-v2` (историческое имя сохранено для обратной совместимости).
- Серверной БД в проекте нет.
- При импорте JSON текущее локальное состояние полностью заменяется.

Важно:
- Если очистить данные браузера, локальная база будет удалена.
- Для безопасности используйте регулярный экспорт JSON в разделе `Данные`.

## Формат JSON для импорта

Импорт поддерживает два варианта:

1. Обертка экспорта:

```json
{
  "schemaVersion": 1,
  "exportedAt": "2026-03-22T00:00:00.000Z",
  "appData": {
    "tags": [],
    "accounts": [],
    "snapshots": [],
    "digestSources": [],
    "digestPosts": []
  }
}
```

2. Чистый объект `appData`:

```json
{
  "tags": [],
  "accounts": [],
  "snapshots": [],
  "digestSources": [],
  "digestPosts": []
}
```

Примечание:
- `digestSources` и `digestPosts` сохраняются в схеме для совместимости, даже если в текущем UI они не используются.

## Деплой на Vercel

1. Создайте репозиторий на GitHub и запушьте проект.
2. В Vercel нажмите `Add New Project` и выберите репозиторий.
3. Framework Preset: `Next.js`.
4. Build Command: `npm run build`.
5. Install Command: `npm install`.
6. Нажмите `Deploy`.

После деплоя приложение будет работать без отдельной БД, так как данные сохраняются в браузере пользователя.

## Ограничения текущей версии

- Нет транзакционного учета и банковских интеграций.
